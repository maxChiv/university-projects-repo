Completed Work

Feedback Integration

Stakeholder Feedback:

  • Website positively received, Frank complimented the look and feel of the site.

  • Request for a document of features we did not implement but how we planned to.

  • Excitement to start using the website.

New Features / Changes

Admin Settings Page

  • Fetches and displays current values after entering the respective initial ‘code’

  • Saves new values correctly and then displays those new values as current values

  • Toast popups (including user friendly error messages for invalid entries)

Costing Summary and Pricing page

  • Split into two easy to navigate sections as per Franks feedback from Sprint 2

  • Uses dynamic form which made it very easy to display values

Project Details Page

  • Must enter correct start and end years to continue across to other tabs

  • Easy to understand navigation between tabs

Testing

  • API Testing

  • Frontend Testing

Not Completed Work

  • Handover documentation and client handover meeting review.

Key Learnings

  • Integration took longer than anticipated which pushed deployment back very close to deadline

  • Some tasks lacked clear acceptance criteria, leading to rework in design discussions.

  • Have far stricter deadlines for tasks.

  • More communication between front and back end throughout sprint rather than just at integration

Stakeholder Notes / Feedback

  • Praise for usability of website.

  • Easy to navigate and understand where data is required to be filled

Next Steps

  • Finalise handover with client
