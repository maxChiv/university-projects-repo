Group Twenty Two Project : CD + Deployment

Created by Eugene Yap, last modified by Zac Davidson on Oct 29, 2025

Overview

The RCPT application consists of four main components:

  1. Frontend – React-based interface for user interaction.

  2. Backend – Spring Boot REST API handling logic and database integration.

  3. PostgreSQL – relational database for reference/lookup data.

  4. MongoDB (Atlas) – NoSQL database storing main app data (users, projects, documents).

Deployment is managed through Render, which builds and hosts each service as a containerised application using Docker. Continuous Deployment (CD) is handled automatically via Git integration.

Project Link: https://group22-rcpt.onrender.com

Backend Deployment (Spring Boot + PostgreSQL + MongoDB Atlas)

Render Service Type:

  • Service Type: Web Service

  • Environment: Docker

  • Branch: main

  • Port: 8080

Environment Variables:

  • SPRING_DATASOURCE_URL → PostgreSQL connection string

  • SPRING_DATASOURCE_USERNAME → PostgreSQL username

  • SPRING_DATASOURCE_PASSWORD → PostgreSQL password

  • MONGO_URI → MongoDB Atlas connection URI

  • FIREBASE_CREDENTIALS_PATH → classpath:firebase-service-account.json

Notes:

  • Render builds the Docker image, caches dependencies, and runs the container.

  • Logs are viewable in Render’s dashboard.

Frontend Deployment (React / Node 22)

Render Service Type:

  • Service Type: Web Service

  • Environment: Docker

  • Branch: main

  • Port: 3000

Environment Variables:

  • VITE_FIREBASE_API_KEY → Firebase API key

  • VITE_FIREBASE_AUTH_DOMAIN → Firebase auth domain

  • VITE_FIREBASE_PROJECT_ID → Firebase project ID

  • VITE_FIREBASE_MESSAGING_SENDER_ID → Firebase messaging sender ID

  • VITE_FIREBASE_APP_ID → Firebase app ID

Databases

PostgreSQL (Render Managed)

  • Created via Render Dashboard → PostgreSQL

  • Render injects DATABASE_URL automatically

  • Map DATABASE_URL to SPRING_DATASOURCE_URL

MongoDB Atlas

  • Hosted externally at https://www.mongodb.com/cloud/atlas

  • Connection string stored as MONGO_URI

Continuous Deployment (CD) on Render

  • GitHub Integration: Render installs a webhook on the repo.

  • Push to main branch triggers automatic deployment.

  • Build process:

    • Pull latest code

    • Build Docker image

    • Start container and perform health check

  • Zero-Downtime Deployment:

    • New container is started

    • Once healthy, traffic is switched to the new version

    • Old container is stopped

  • Logs and Rollbacks:

    • Viewable in Render Dashboard

    • Rollback to previous deploy with one click

Security Considerations

  • Never commit credentials (.env, service accounts) to Git

  • Use Render Environment Variables for all secrets

  • Restrict MongoDB Atlas access to Render static outbound IPs

  • Use HTTPS for all frontend-backend communication
