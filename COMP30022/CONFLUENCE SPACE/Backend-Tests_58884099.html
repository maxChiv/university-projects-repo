Group Twenty Two Project : Backend Tests

Created by Mitchell Pooley on Oct 17, 2025

Overview

Our backend code includes automated integration tests using JUnit. Tests support CRUD (Create, Read, Update, DELETE) operations, REST API mocking, persistence of data and check that JPA and H2 databases work as expected.

Tests

Each controller test consists of the same structure:

  • testGetAll()

  • TestGetBy…()

  • TestPost()

  • TestDelete()

Each creates data via JPA and pushes said data to an H2 database. By using H2, we can ensure that the system correctly hits the database without hitting the real database and risking affecting real data. Depending on the test, different CRUD operations are then performed by using MockMVC to mock a REST API to simulate real HTTP requests and hit the relevant endpoint.

Running the Tests

  • Tests can be run locally by running ./mvnw clean test from the /Backend directory.

  • Alternatively, tests are run automatically when making a pull request or when pushing to dev and main branches. This is achieved via the CI workflow
