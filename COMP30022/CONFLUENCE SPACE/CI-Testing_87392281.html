Group Twenty Two Project : CI Testing

Created by Eugene Yap on Oct 29, 2025

This project uses GitHub Actions for continuous integration. The workflow is located at .github/workflows/ci.yml.

The CI pipeline runs automatically on every push or pull request to the main and dev branches. It verifies that both the frontend and backend build correctly and that tests and checks pass before any changes are merged.

There are two main jobs in the workflow:

  1. Frontend – Build and Type Check

  2. Backend – Unit and Integration Tests

The workflow is triggered when code is pushed to or a pull request is opened against the main or dev branches. It runs with read-only permissions to the repository.

The first job ensures that the frontend compiles successfully and all type checks pass. It runs on an Ubuntu environment using Node.js version 22 and operates in the Frontend directory.
The steps performed in this job include:

  • Checking out the repository to access the project files.

  • Setting up the Node.js environment and caching dependencies for faster builds.

  • Installing frontend dependencies using the project’s package manager.

  • Running type checking to catch any type errors in the codebase.

  • Building the frontend application to confirm it compiles successfully.

The second job, called Backend – Unit and Integration Tests, verifies that all backend tests pass. It runs on an Ubuntu environment using Java version 17 (Temurin distribution) and operates in the Backend directory.
The steps performed in this job include:

  • Checking out the repository to access the project files.

  • Setting up the Java environment and enabling Maven dependency caching.

  • Making the Maven wrapper executable to ensure tests can be run.

  • Executing all unit and integration tests to confirm backend functionality.

Both jobs run independently and in parallel. If either one fails, the workflow will fail. Successful completion of the CI process means the frontend builds without errors, type checks pass, and all backend tests run successfully.

This workflow helps maintain code quality and prevents broken builds from being merged into main or dev.
