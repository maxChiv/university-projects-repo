# Dynamic Form System Implementation Specification

## Overview

Build a **generalized JSON-driven form system** that can render any form from configuration files. The current `ProjectFormDemo` will serve as a **reference example**, but the system must work for **any form** (staff costs, equipment, budgets, settings, etc.).

## Goal

Create a reusable form infrastructure where:

1. **Any form is defined by a JSON file** - no code changes needed
2. **One `DynamicForm` component renders all forms** - handles any field type
3. **Validation rules live in JSON** - converted to Zod at runtime
4. **New forms = new JSON file only** - no new components needed

---

## Core Architecture (3 Parts)

### 1. JSON Schema Files (`/public/forms/*.json`)

**Purpose:** Define form structure, fields, validation

**Example:** `projectForm.json`, `staffForm.json`, `budgetForm.json`

**Schema structure:**

```json
{
  "formId": "unique-form-identifier",
  "submitLabel": "Submit Button Text",
  "fields": [
    {
      "name": "fieldName",
      "type": "text|textarea|number|date|select|checkbox|repeatable",
      "label": "Display Label",
      "placeholder": "Placeholder text (optional)",
      "message": "Help text below field (optional)",
      "validation": {
        "required": true|false,
        "minLength": number,
        "maxLength": number,
        "min": number,
        "max": number,
        "pattern": "regex string",
        "email": true|false,
        "url": true|false
      },
      "options": [{"value": "...", "label": "..."}],  // for select fields
      "prefix": "$",  // for number fields
      "fields": [...],  // for repeatable sections
      "defaultEntry": {}  // for repeatable sections
    }
  ]
}

```

**Key principle:** JSON is self-contained - has everything needed to render and validate the form.

### 2. Zod Schema Builder (`/src/utils/zodSchemaBuilder.ts`)

**Purpose:** Convert JSON validation rules into Zod schemas at runtime

**Core function:**

```tsx
export function buildFormSchema(fields: FieldDefinition[]): z.ZodObject<any>

```

**What it does:**

- Takes array of field definitions from JSON
- For each field, generates appropriate Zod type based on `field.type`
- Applies validation constraints from `field.validation`
- Returns complete Zod schema object

**Validation mapping:**

| JSON Rule | Zod Implementation |
| --- | --- |
| `"required": true` | Remove `.optional()` |
| `"minLength": 5` | `.min(5, "message")` |
| `"maxLength": 100` | `.max(100, "message")` |
| `"min": 0` | `.min(0, "message")` (for numbers) |
| `"max": 1000` | `.max(1000, "message")` (for numbers) |
| `"pattern": "^[A-Z]"` | `.regex(new RegExp(...))` |
| `"email": true` | `.email("message")` |
| `"url": true` | `.url("message")` |

**Example implementation pattern:**

```tsx
function buildFieldSchema(field: FieldDefinition): z.ZodTypeAny {
  let schema: z.ZodTypeAny;

  switch (field.type) {
    case "text":
      schema = z.string();
      if (field.validation?.minLength) {
        schema = schema.min(field.validation.minLength, `Min ${field.validation.minLength} chars`);
      }
      if (field.validation?.email) {
        schema = schema.email("Invalid email");
      }
      break;

    case "number":
      schema = z.number({invalid_type_error: "Must be a number"});
      if (field.validation?.min !== undefined) {
        schema = schema.min(field.validation.min, `Min value: ${field.validation.min}`);
      }
      break;

    // ... other types
  }

  // Apply optional unless required
  if (!field.validation?.required) {
    schema = schema.optional();
  }

  return schema;
}

```

### 3. DynamicForm Component (`/src/components/forms/DynamicForm.tsx`)

**Purpose:** Universal form renderer that works with any JSON schema

**Props interface:**

```tsx
type DynamicFormProps = {
  schema: FormSchema;                    // Loaded JSON
  onSubmit: (data: any) => Promise<void>; // Submit handler
  initialData?: any;                      // For edit mode
  isLoading?: boolean;                    // Loading state
};

```

**Rendering logic:**

```tsx
export function DynamicForm({ schema, onSubmit, initialData, isLoading }: DynamicFormProps) {
  const [saving, setSaving] = useState(false);

  // 1. Generate Zod schema from JSON
  const zodSchema = buildFormSchema(schema.fields);

  // 2. Initialize React Hook Form
  const form = useForm({
    resolver: zodResolver(zodSchema),
    defaultValues: initialData || {}
  });

  // 3. Handle submit
  const handleSubmit = async (values: any) => {
    setSaving(true);
    try {
      await onSubmit(values);
    } finally {
      setSaving(false);
    }
  };

  if (isLoading) return <p>Loading form...</p>;

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(handleSubmit)}>
        {schema.fields.map(field => (
          <FieldRenderer key={field.name} field={field} control={form.control} />
        ))}

        <Button type="submit" disabled={saving}>
          {saving ? "Saving..." : schema.submitLabel}
        </Button>
      </form>
    </Form>
  );
}

```

**Field renderer (part of DynamicForm or separate component):**

```tsx
function FieldRenderer({ field, control }) {
  switch (field.type) {
    case "text":
      return <FormTextInput control={control} {...field} />;
    case "textarea":
      return <FormTextArea control={control} {...field} />;
    case "number":
      return <FormNumberInput control={control} {...field} />;
    case "date":
      return <FormDateInput control={control} {...field} />;
    case "select":
      return <FormSelect control={control} {...field} />;
    case "checkbox":
      return <FormCheckbox control={control} {...field} />;
    case "repeatable":
      return <FormRepeatableArray control={control} {...field} />;
    default:
      console.warn(`Unknown field type: ${field.type}`);
      return null;
  }
}

```

## Page Implementation Pattern

**Any page that needs a form follows this pattern:**

```tsx
// Example: /src/pages/StaffCostsPage.tsx
export function StaffCostsPage() {
  const [schema, setSchema] = useState(null);
  const [initialData, setInitialData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    async function loadForm() {
      // 1. Load JSON schema
      const response = await fetch("/forms/staffCosts.json");
      const formSchema = await response.json();
      setSchema(formSchema);

      // 2. Optionally load existing data (for edit mode)
      // const data = await StaffService.getData();
      // setInitialData(data);

      setLoading(false);
    }
    loadForm();
  }, []);

  async function handleSubmit(data: any) {
    // 3. Save data via appropriate service
    await StaffService.saveData(data);
    console.log("Saved:", data);
  }

  if (loading || !schema) return <p>Loading...</p>;

  return (
    <div className="container">
      <h1>Staff Costs</h1>
      <DynamicForm
        schema={schema}
        onSubmit={handleSubmit}
        initialData={initialData}
      />
    </div>
  );
}

```

**This same pattern works for:**

- Project info forms
- Budget forms
- Equipment forms
- User settings forms
- **Any form you need**

## Implementation Tasks

### Phase 1: Foundation

**Task 1.1** - Create TypeScript types for JSON schema structure

- File: `/src/types/FormSchema.ts`
- Define: `FieldDefinition`, `ValidationRules`, `FormSchema`

**Task 1.2** - Build Zod schema builder utility

- File: `/src/utils/zodSchemaBuilder.ts`
- Implement: `buildFieldSchema()`, `buildFormSchema()`
- Support all field types: text, number, date, select, checkbox, repeatable
- Support all validation rules: required, min/max, patterns, email, url

### Phase 2: Components

**Task 2.1** - Create DynamicForm component

- File: `/src/components/forms/DynamicForm.tsx`
- Accept schema prop
- Generate Zod schema
- Render fields dynamically
- Handle submit

**Task 2.2** - Create/verify field renderer

- Either inline in DynamicForm or separate `FieldRenderer` component
- Map field types to existing wrapper components

### Phase 3: Example Implementation

**Task 3.1** - Create JSON schema from ProjectFormDemo

- File: `/public/forms/projectForm.json`
- Convert current hard-coded form to JSON structure
- Include all fields, validation, options

- File: `/src/pages/ProjectFormPage.tsx` (or update existing)

## Success Criteria

✅ **Generic system** - DynamicForm works with any JSON schema, not just project form

✅ **JSON-driven** - Adding new form = write JSON file, no code changes

✅ **Validation flexible** - All validation rules configurable via JSON

✅ **Type-safe** - TypeScript catches schema structure errors

✅ **Reuses components** - All existing field wrappers (`FormTextInput`, etc.) work unchanged

✅ **Service agnostic** - Works with any data service (Project, Staff, Budget, etc.)

✅ **Easy to extend** - New field types = add case to renderer + create wrapper component

## Notes

- **Current `ProjectFormDemo` is reference only** - system must be generic
- **Don't optimize for one form** - think "how does this work for 20 different forms?"
- **JSON is the API** - think of it like a contract that could come from backend
- **Keep existing components** - `FormTextInput`, `FormSelect`, etc. are already built and work
- **Service layer unchanged** - `ProjectService` doesn't know forms are dynamic
- **Future proof** - Should work when JSON comes from API instead of static files
