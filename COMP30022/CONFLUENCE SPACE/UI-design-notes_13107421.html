1. High-Level Information Architecture (Pages)

From the requirements, we see the following main pages:

  1. Login / Landing Page

    • UoM branding

  2. Dashboard

    • List of projects

    • "Create New Project" button

    • Notifications panel (pending approvals, comments, deadlines)

    • Admin-only view: system-wide visibility, update salary rates, manage workflows

    • Approvals Workflow

      • Step-tracker (Draft → HoD → Dean → RIC → Approved)

      • Secure read-only approval view (shared link for approvers)

      • Approve / Reject / Request Changes buttons

      • Comment block

  3. Project Setup / Summary Page

    • Project details form (title, duration, faculty, funding type, attachments) stuff from excel

    • Project status indicator (draft → submitted → approved)

    • Inline validation (must complete required fields before continuing)

    • Budget Summary View

      • Consolidated view of all cost and price data

      • Different "modes" (Budget Form, Oncost Calculator, TR1F if applicable), not completely necessary, but could be good to have different summary views.

      • Highlight non-compliance (e.g., failing multiplier check)

      • Export buttons (PDF, Excel)

  4. costing summary pages (with adding functionality)

    • Staff Costs

      • Add staff member button

      • collapsible and expandable table oof summarised information

      • Timeline view for staff allocation (in expandable version)

      • Rolling cost total visible on sidebar??

      • In-kind contributions section??

      • group by type for clarity

    • Non-Staff Costs

      • Consumables, travel, equipment, services

      • Group by type for clarity ?

    • Oncosts View ????

      • Automatic calculation with editable overrides

      • Clear summary panel

  5. adding page

  • add in a costing page for both types of costs

  • this is then adjustable and extendbile for a type of cost, or a type of entry

  1. Pricing & Adjustments (could be placed inside the summary page as an editing functionality for these things)

    • Cost vs Price comparison

    • Margin/discount input fields (sliders or % inputs)

    • University’s recommended vs user-entered multipliers

    • Real-time recalculation display

  2. Reports & Exports

    • Choose export format (Budget form PDF, Offer Letter PDF, CSV, bulk export for admins)

    • History of exports (audit logging)

  3. Admin Panel (for RIC)

    • Manage salary/indirect rates

    • Configure workflow steps

    • Dashboard with overview of all projects and statuses

    • Export multiple reports

2. Core User Flows

Flow A: Researcher creating a project

  1. Dashboard → New Project

  2. add in the details of the project from the adding form thats tailored for this.

  3. Fill in Project Summary (basic details)

  4. Saves this project to dashboard, and now editable

Flow B: Approver reviewing a project

  1. Email notification → Approval link (or login → dashboard)

  2. Open project notification bell

  3. See Budget Summary + step tracker

  4. Leave comments (if any)

  5. Approve / Request changes

Flow C: Admin updating system rates

  1. Login → Admin Dashboard

  2. Navigate to System Configuration

  3. Update salary/indirect cost rates

  4. Save → changes reflected in all new projects

3. Wireframe Sketch Concepts (in words)

Think of these as modular views with a consistent left-nav:

  • Left Sidebar Navigation XXXX

    • Dashboard

    • Project Summary

    • Staff Costs

    • Non-Staff Costs

    • Approvals

    • Reports

  • Top Header Bar

    • Project title (editable inline)

    • Status indicator (draft/submitted/approved)

    • Export button

    • Notifications (bell icon with comments/approval requests)

  • Right Sidebar (Context Panel)

    • Inline help/tooltips

    • Running totals (cost/price, compliance alerts)

    • Contextual glossary definitions

Each cost-entry screen (Staff, Non-Staff) could be a table-like form with:

  • Rows for entries (like Excel, but simpler)

  • “+ Add item” button

  • Totals updating in sidebar
