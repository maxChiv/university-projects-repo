Meeting Notes

Frank’s Feedback on Figma:

  • “Other Costs” can be simplified into Staff and Non-Staff categories.

  • Grouping categories is important for the budget form.

  • Need a Cost Summary vs Price Summary page (suggest calling it Cost Summary and Pricing).

Design Changes:

  • Allow flexibility for:

    • Adding discounts (lump sum or percentage).

    • Adding margins for individual cost items.

  • Only admins should modify these values; end-users cannot override.

  • In-kind contributions do not involve pricing.

  • Design inspiration aligns with existing UniMelb tools → confirmation we are on the right track.

New Features / Changes:

Summary Page:

  • Separate pages for Cost Summary and Pricing.

  • Users can review costs and plan ways to reduce them.

  • Cost recovery multiplier: 1.5 → applies to total project cost, only visible to admin, not users.

Approval Flow:

  • Approvers receive:

    • Email notification.

    • PDF view of the project (for review).

  • Users can add additional viewers (e.g., supervisors, RIC staff).

  • If the project is edited after approval:

    • Approval emails must be resent.

    • There should be a clear staging process for edits.

Budget & Emails:

  • Add CC option for multiple viewers in export form.

  • Ensure only one approver but allow multiple viewers.

Other Notes:

  • Simplify cost categories: Staff and Non-Staff only.

  • On the summary page: show all costs and underneath ask “How will you modify this?”

  • Add clear descriptions to improve clarity.

  • Clarify what message approvers receive if the budget changes.
