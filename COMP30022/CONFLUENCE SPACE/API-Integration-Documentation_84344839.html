Overview

This implementation creates a complete bridge between your Spring Boot backend (port 8080) and React frontend (port 3000/5173). The connection uses REST API calls over HTTP.

How The Connection Works - Step by Step

🔄 Complete Request Lifecycle

Let's trace a single API call from button click to data display:

1. USER CLICKS "Create Project" BUTTON
   └─► React Component (ProjectDashboard.tsx)

2. COMPONENT CALLS SERVICE
   └─► projectService.create({ title: "New Project" })

3. SERVICE PREPARES REQUEST
   └─► Adds base URL: http://localhost:8080/api/v1/projects
   └─► Sets headers: Content-Type: application/json
   └─► Converts data to JSON

4. FETCH API SENDS HTTP REQUEST
   └─► POST http://localhost:8080/api/v1/projects
   └─► Body: {"title": "New Project"}

5. SPRING BOOT RECEIVES REQUEST
   └─► CORS filter allows the request (from localhost:3000)
   └─► ProjectController.create() method executes
   └─► Saves to MongoDB
   └─► Returns JSON response

6. FRONTEND RECEIVES RESPONSE
   └─► api.ts handleResponse() processes it
   └─► Converts JSON to JavaScript object
   └─► Returns to service layer

7. SERVICE RETURNS TO COMPONENT
   └─► Component receives the new project object
   └─► Updates state
   └─► Re-renders UI with new data

🔍 Detailed Code Flow

// 1. Component initiates action
const handleCreateProject = () => {
  projectService.create(data, userId)  // Calls service
}

// 2. Service formats the request
projectService.create = (data, userId) => {
  return apiService.post('api/v1/projects', data, {
    headers: { 'X-User-Id': userId }
  })
}

// 3. API service sends HTTP request
apiService.post = (path, data) => {
  return fetch('http://localhost:8080/api/v1/projects', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(data)
  })
}

// 4. Backend processes (Java)
@PostMapping
public ResponseEntity<ProjectResponse> create(@RequestBody ProjectCreateRequest req) {
  Project saved = service.create(req);
  return ResponseEntity.ok(saved);
}

// 5. Frontend processes response
handleResponse = (response) => {
  if (!response.ok) throw new Error();
  return response.json();  // Parse JSON
}

// 6. Component updates UI
.then(project => {
  setProjects([...projects, project]);  // Add to list
  setShowSuccess(true);                 // Show success message
})

What Each File Does

Frontend Files (React)

1. config.ts - Configuration Center

// What it does:
- Reads environment variables (VITE_API_BASE_URL)
- Determines if app is in development or production
- Provides base URL for all API calls
- Sets default headers for requests

// Why it's needed:
- Centralizes configuration
- Makes it easy to switch between dev/prod
- Prevents hardcoded URLs throughout the app

2. api.ts - HTTP Request Handler

// What it does:
- Sends actual HTTP requests using fetch()
- Handles all HTTP methods (GET, POST, PUT, DELETE)
- Processes responses and errors
- Adds authentication headers (when implemented)

// Key functions:
- handleResponse(): Parses JSON, handles errors
- apiRequest(): Core function that makes the fetch call
- apiService.get/post/put/delete: Convenience methods

// Error handling:
- Catches network errors
- Parses error messages from backend
- Throws standardized errors for components

3. projectService.ts - Project-Specific Logic

// What it does:
- Provides functions for all project operations
- Maps backend DTOs to frontend types
- Handles project-specific business logic

// Example:
projectService.create() → calls api.post() → returns typed Project object

4. useApi.ts - React Hooks

// What it does:
- Manages loading states automatically
- Handles errors consistently
- Provides refetch functionality
- Implements pagination logic

// Key hooks:
- useApi(): Fetches data on component mount
- useMutation(): Handles create/update/delete
- usePagination(): Manages paginated lists

5. ProjectDashboard.tsx - UI Component

// What it does:
- Renders the user interface
- Uses hooks to fetch data
- Handles user interactions
- Updates UI based on API responses

Backend Files (Spring Boot)

1. CorsSecurityConfig.java - CORS Configuration

// What it does:
- Allows frontend (localhost:3000) to access backend
- Specifies allowed HTTP methods
- Sets allowed headers
- Without this, browser blocks the requests

2. ProjectController.java - REST Endpoints

// What it does:
- Receives HTTP requests
- Validates input data
- Calls service layer
- Returns JSON responses

Troubleshooting

Common Issues and Solutions:

1. "CORS Error"

Error: Access to fetch at 'http://localhost:8080/api/v1/projects' from origin 'http://localhost:3000' has been blocked by CORS policy

Solution:

  • Verify backend CorsSecurityConfig includes your frontend URL

  • Make sure backend is running

  • Check if you're using the correct ports

2. "Connection Refused"

Error: Failed to fetch - net::ERR_CONNECTION_REFUSED

Solution:

  • Backend is not running → Start with mvn spring-boot:run

  • Wrong port → Check VITE_API_BASE_URL in .env

3. "404 Not Found"

Error: API Error 404: Not Found

Solution:

  • Wrong endpoint URL → Check the path in service files

  • Missing /api/v1/ prefix in the URL

4. "No Data Showing"

Solution:

  • Check browser DevTools Network tab

  • Verify response contains data

  • Check console for JavaScript errors

  • Ensure state is updating correctly

Security Considerations

Current Implementation:

  • ❌ No Authentication - Anyone can access the API

  • ❌ No Authorization - No role-based access control

  • ✅ CORS Configured - Only specified origins can access

  • ✅ Input Validation - Backend validates data

For Production:

  1. Add JWT Authentication

  2. Implement HTTPS

  3. Add Rate Limiting

  4. Validate all inputs

  5. Sanitize data before displaying

  6. Use environment variables for sensitive data
