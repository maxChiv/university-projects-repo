## Software Modelling and Design (SWEN30006)

### Course Description
[Software Modelling and Design, University of Melbourne Handbook](https://handbook.unimelb.edu.au/2025/subjects/swen30006)

Aim is to construct a software system, requirements must be analysed and modelled, and designs developed and evaluated; this subject teaches knowledge and skills needed for these tasks. This includes the development of static and dynamic models for aspects of both the problem space and the solution space. The emphasis here is on an Agile approach, and on techniques appropriate for object-oriented development.

### Assignments Overview

- **Assignment 1: Droning On**
  - This project involves extending the existing drone-based parcel delivery simulation, "Droning On," developed by Delivering Solutions Inc., to improve handling of fragile and heavy parcels and enhance access control for drones navigating suburban streets. The current system simulates drones flying only above roads, with movement controlled by a first-come, first-served access policy for locations. The required enhancements include implementing slower flight for fragile parcels to reduce breakage, caching and prioritizing location access requests based on parcel fragility, weight, and drone position, and preserving all existing simulation behaviour where no special conditions apply. The task includes designing and implementing these features within the provided Java codebase, maintaining compliance with the simulation’s time-step model, and producing documentation explaining design choices with supporting diagrams. The solution must build and run from the command line and pass automated testing.

- **Assignment 2: Specification for Pinochle**
  - This project involves redesigning and extending a digital version of the Pinochle card game developed by CardCraft Studio to improve its scalability, maintainability, and feature set. The game, played between two players using a special 48-card deck, includes phases such as dealing, bidding, melding, trick-taking, and scoring, with the computer player initially employing random strategies. The task is to enhance the system by adding support for additional melds, implementing smarter bidding and trick-taking AI, and introducing a cut-throat gameplay mode. The project requires modifying the provided IntelliJ-based Java codebase, ensuring automated test cases pass, maintaining existing game functionality, and producing design documentation explaining the design choices and AI strategies.
